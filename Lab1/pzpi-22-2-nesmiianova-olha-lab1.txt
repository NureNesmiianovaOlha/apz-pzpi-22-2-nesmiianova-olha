2 ЗАВДАННЯ


     Розробити документ Vision and Scope для програмної системи для контролю за успішністю студентів коледжу. Ця система повинна забезпечити ефективний моніторинг оцінок студентів, їхнього прогресу, а також автоматизувати процеси підготовки звітів для викладачів та адміністрації. У рамках завдання передбачено підготовку фінального документа у форматі PDF, а також експорт його у текстовий файл з кодуванням UTF-8 для подальшого розміщення у репозиторії на GitHub.
     
3 ОПИС ВИКОНАНОЇ РОБОТИ


     Під час виконання роботи було визначено ключові вимоги до програмної системи контролю за успішністю студентів коледжу. Проаналізувавши вже існуючі рішення у сфері управління студентськими оцінками дозволило нам виявити основні функціональні обмеження таких систем, як «E-Learning» та «MyCollege», а також розробити ідеї для вдосконалення нашої платформи. Основні функції програми охоплюють: введення оцінок та ведення електронного журналу, створення звітів про успішність студентів, аналіз успішності на основі зібраних даних, можливість фільтрації та сортування результатів, інтерфейс для викладачів та адміністрації для додавання та редагування оцінок.
     На основі цього було розроблено документ Vision and Scope, що деталізує концепцію програмної системи та її основні функціональні блоки.
    
4 ВИСНОВОК 

     У результаті виконання лабораторної роботи буде розроблено Vision and Scope, який допоможе визначити функціональність програмної системи для контролю за успішністю студентів коледжу, яка дозволить зручно вводити та зберігати дані про оцінки, а також автоматизує процес підготовки звітів. Система
допомагає зменшити навантаження на викладачів та адміністрацію, забезпечуючи зручний інтерфейс для управління оцінками та успішністю студентів. Виконання лабораторної роботи сприятиме розвитку практичних навичок у програмуванні, розробці бази даних та створенні інтерфейсу користувача.

1.	Бізнес-вимоги
     1.1.	Передумови
     У сучасних умовах освіти важливість контролю за успішністю студентів зростає. Традиційні методи обробки та зберігання даних про студентів часто є трудомісткими і схильні до помилок, що негативно впливає на швидкість прийняття рішень. Наявність централізованої системи, яка дозволяє автоматизувати процес контролю за успішністю студентів, значно покращить точність даних, зменшить навантаження на викладачів та забезпечить швидкий доступ до актуальної інформації. Студент з легкістю зможе отримати інформацію щодо своїх успіхів у навчанні, проаналізувати свої оцінки  та отримати звіт за фільтрованими параметрами, що дозволяє покращити самоосвіту та своєчасно реагувати на можливі проблеми в навчальному процесі.
1.2.	Бізнес-можливість
     На ринку вже існують рішення для контролю за успішністю студентів, такі
     як системи електронного оцінювання та онлайн-реєстрації для університетів та коледжів. Ці платформи дозволяють викладачам зберігати та обробляти оцінки студентів, але часто обмежуються лише базовими функціями, такими як введення оцінок або перегляд академічного прогресу. Крім того, вони не завжди забезпечують зручний доступ до даних для студентів і викладачів, а також не пропонують інтуїтивно зрозумілу аналітику для відстеження динаміки успішності. Наше рішення має на меті значно покращити цей процес, надаючи платформу з більш широкими можливостями. Програмна система для контролю за успішністю студентів коледжу дозволить автоматизувати всі аспекти контролю за навчальними досягненнями: від введення оцінок і складання звітів до моніторингу прогресу студентів через інтерактивну аналітику. Користувачі (як студенти, так і викладачі) зможуть швидко і зручно отримувати актуальну інформацію про успішність, аналізувати дані за різними параметрами.
1.3.	Способи монетизації 


     - Плата за підписку: Навчальні заклади можуть отримати доступ до системи через підписку. Цей спосіб монетизації забезпечить постійний дохід від системи.
     - Плата за інтерфейс користувача та аналітику: Платформа може пропонувати додаткові платні модулі для створення глибших звітів, аналізу та статистики, які допомагають адміністрації навчальних закладів приймати стратегічні рішення.
     - Реклама на платформі: Програмна система може впровадити рекламу, пов'язану з освітніми послугами, курсами або навчальними матеріалами, що дозволить додатково генерувати дохід.
       


2.	Концепція рішення
2.1.	Окреслення концепції
     Метою розробки програмної системи є створення ефективного та зручного інструменту для автоматизації процесу контролю за успішністю студентів у коледжі. Система повинна спростити процес збору, обробки та збереження інформації про оцінки, відвідуваність та інші важливі показники успішності студентів. Вона надасть викладачам і адміністрації зручні інструменти для введення даних, їх аналізу та формування звітів, а студентам — можливість переглядати свої оцінки та результати за різні періоди. Основною метою є спрощення і автоматизація процесу, що дозволить викладачам зосередитися на навчальному процесі, а адміністрації — на управлінні навчальними даними. Система також забезпечить прозорість оцінювання, що сприятиме підвищенню ефективності навчального процесу.
2.2.	Головна функціональність

     Функції, що будуть виконуватися системою контролю за успішністю студентів коледжу:
     MF 1) Введення оцінок. Викладачі можуть вводити оцінки студентів за різними дисциплінами та семестрами. Для кожного студента передбачено інтерфейс для вводу його результатів у вигляді числових оцінок або коментарів. Всі введені дані автоматично зберігаються та доступні для перегляду адміністратором та студентами.
     MF 2) Управління відвідуваністю. Викладачі можуть відмічати відвідуваність студентів за кожне заняття. Ця інформація зберігається в системі для подальшого аналізу та використовується при підрахунку загального успіху студента.
     MF 3) Генерація звітів. Адміністрація та викладачі можуть створювати звіти про успішність студентів на основі введених даних. Звіти можуть бути скомпільовані по групах, дисциплінах, факультетах, а також за індивідуальними студентами. Це дозволить легко відстежувати прогрес студентів та вчасно реагувати на проблеми.
     MF 4) Фільтрація та сортування результатів. Система надає можливість фільтрувати оцінки та інші дані по різних параметрах: за курсами, групами, дисциплінами, періодами часу. Це дасть змогу швидко знаходити потрібну інформацію, зручним способом аналізувати успішність студентів та вчасно коригувати навчальні процеси.
     MF 4) Перегляд результатів студентами. Студенти зможуть переглядати свої оцінки, звіти про успішність та інші результати навчання в особистому кабінеті. Всі дані будуть оновлюватися в реальному часі, що дозволить студентам відстежувати свій прогрес і планувати подальшу навчальну діяльність.
     MF 5) Інтерфейс для викладачів. Викладачі матимуть доступ до введення оцінок, відвідуваності та перегляду звітів по своїх групах.
     MF 6) Мобільна версія для студентів і викладачів. Враховуючи потребу в мобільності, планується створення мобільної версії системи, яка дозволить студентам та викладачам доступати до даних через смартфони або планшети. Це дасть змогу бути в курсі результатів навчання в будь-який час та з будь-якого місця.
     


    3.	Рамки та обмеження
     3.1.	Рамки первинного випуску
      
     Перелік функціональності для програмної системи контролю за успішністю студентів коледжу:
     - MF 1-1) Зберігання інформації про студентів, їх оцінки, відвідуваність та інші показники успішності.
     - MF 1-2) Введення оцінок та відміток про відвідуваність від викладачів.
     - MF 1-3) Формування звітів про успішність студентів по групах, дисциплінах та семестрах.
     - MF 1-4) Перегляд студентами своїх оцінок та результатів через особистий кабінет.
     - MF 1-5) Реєстрація викладачів та адміністрації, а також налаштування прав доступу до даних.
     - MF 1-6) Захист даних студентів та викладачів шляхом автентифікації та шифрування даних.

3.2.	Рамки наступних випусків

     Перелік функціональності для веб та мобільних клієнтів програмної системи контролю за успішністю студентів:
     - MF 2-1) Інтеграція з системами електронного навчання для автоматичного оновлення оцінок та відвідуваності.
     - MF 2-2) Надсилання сповіщень через email або push-повідомлення для студентів про зміни в оцінках, тощо.
     - MF 2-3) Створення аналітичних звітів про успішність студентів на основі їхніх оцінок та відвідуваності.
     - MF 2-4) Мобільна версія для студентів та викладачів, що забезпечить доступ до результатів та звітів на смартфонах.
     - MF 2-5) Додавання можливості перегляду історії успішності студентів та генерації звітів для викладачів та адміністрації.


4.	Бізнес-контекст

4.3 Технології, які будуть використані у розробці


     Система буде використовуватись викладачами, студентами та адміністрацією коледжу, які можуть знаходитися в різних регіонах та часових зонах. Основним завданням є забезпечення доступу до системи в будь-який час доби для введення оцінок, перегляду результатів успішності, управління даними та створення звітів. Система повинна бути доступною через веб-платформу та мобільний додаток для зручності користувачів, забезпечуючи високий рівень надійності, безпеки та продуктивності. Система буде інтегрована з існуючими інформаційними системами навчального закладу та передбачатиме інтеграцію з мобільними додатками для викладачів та студентів. Важливими вимогами є стабільне інтернет-з’єднання та забезпечення безперебійної роботи на всіх етапах користування системою.
     Мови програмування та технології:
     1. JavaScript (React): Використовуватиметься для створення інтерфейсу веб-додатку. React забезпечує створення інтерактивного, швидкого та зручного інтерфейсу для користувачів системи.
     2. Node.js/Express.js: Використовуватиметься для розробки серверної частини, що забезпечує ефективну роботу RESTful API та інтеграцію з базою даних. Це дозволить обробляти запити від користувачів, виконувати обчислення та надавати необхідну інформацію у реальному часі.
     3. Інфраструктура серверної частини:
     - Express: Веб-фреймворк для створення серверної частини. Забезпечує обробку HTTP-запитів, маршрутизацію та підключення middleware.
     - MongoDB: Для зберігання даних про студентів, оцінки, дисципліни та групи. Використання NoSQL бази даних дасть можливість гнучко зберігати та оновлювати інформацію.
     - Mongoose: Бібліотека для роботи з MongoDB, яка дозволяє створювати схеми та зручне управління даними.
     - JWT (JSON Web Token): Система авторизації, яка забезпечує безпечний доступ до різних частин системи для студентів, викладачів та адміністрації.
     - CORS: Middleware для налаштування безпеки та дозволу взаємодії між сервером та клієнтами з різних доменів.
     - Docker: Використовуватиметься для контейнеризації серверної частини, що дозволить спростити процеси розробки та деплойменту додатку.
     4. Інфраструктура фронтенд частини:
     - React: Для створення інтерактивного веб-інтерфейсу.
     - Material-UI: Для стилізації інтерфейсу, забезпечуючи сучасний і зручний дизайн, що підходить для навчальних закладів.
     5. Інфраструктура мобільного додатку:
     - Kotlin: Для розробки мобільного додатку для Android. Це дозволить студентам і викладачам зручно переглядати оцінки, відвідуваність, результати та іншу інформацію через мобільні пристрої.
